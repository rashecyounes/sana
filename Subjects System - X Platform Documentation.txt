Subjects System - X Platform Documentation
Overview

The Subjects system is designed to organize educational content into clear categories.

Each subject represents a main category (e.g., Arabic, English), and each subject can contain multiple courses in later stages.

This allows users to easily navigate and choose the subject they want before accessing courses.

Subject Creation Policy
Subjects are created only by Admin users
Regular users (students or teachers) cannot create or modify subjects
Subjects are managed through the Django Admin Panel
Subject Structure

Each subject contains:

Name
Slug (auto-generated for clean URLs)
Image
Created date
Updated date
Example:
Arabic
English
Database
Table: subjects_subject

Fields:

id
name (unique)
slug (unique)
image
created_at
updated_at
Notes:
slug is automatically generated from the subject name
Used for clean URLs like:
/subjects/arabic/
/subjects/english/
Permissions
Action Who can perform
View subjects Everyone (Public)
Create subject Admin only
Update subject Admin only
Delete subject Admin only
API Endpoints
Get all subjects
GET /api/subjects/
Search subjects (from database)
GET /api/subjects/?search=arabic
Get single subject
GET /api/subjects/{slug}/
Create subject (Admin only)
POST /api/subjects/
Update subject (Admin only)
PATCH /api/subjects/{slug}/
Delete subject (Admin only)
DELETE /api/subjects/{slug}/
Search System
Search is implemented using Django REST Framework
Uses database-level filtering (NOT frontend filtering)
Searches in:
name
Example:
/api/subjects/?search=لغة
