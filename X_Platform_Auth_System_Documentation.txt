X Platform - Authentication and Security System Documentation
Project Overview

X Platform is an e-learning platform designed initially for Jordanian Tawjihi students. The platform currently focuses on Arabic and English subjects, with future expansion support.

Authentication System

The platform uses a custom Django User model.

Users register using:

First name
Last name
Username
Email or phone number
Password
Password confirmation

All newly registered users are automatically assigned the role:

student

Users are NOT allowed to select their role during registration.

Admin users can later update roles via Django Admin:

teacher
admin
User Roles
student
teacher
admin
Password Security

Passwords are never stored as plain text.

Django automatically:

Hashes passwords using PBKDF2
Adds a unique salt per password

This ensures strong protection even if the database is exposed.

SQL Injection Protection

All database operations use Django ORM.

Raw SQL is avoided entirely.

This protects the system from SQL Injection attacks.

JWT Security

Authentication is based on JSON Web Tokens:

Access Token (short-lived)
Refresh Token (long-lived)

Security features:

Refresh Token Rotation enabled
Old tokens are blacklisted after use

This prevents token reuse attacks.

Device Identification

Each login request must include a unique device ID.

Frontend:

Generates device ID
Stores it in localStorage

Headers used:

x-device-id
x-device-name

This allows the backend to track devices and enforce limits.

Active Session Policy (Updated)

Max Devices per user:

2

Max Active Sessions per user:

1
✅ Updated Behavior (Important)

The system now handles sessions intelligently:

1. Same Device Login

If the user logs in again from the SAME device:

→ Old session is automatically deactivated
→ New session is created
→ Login is allowed
2. Different Device Login

If the user tries to log in from another device while a session is active:

→ Login is blocked
🎯 Goal
Prevent account sharing
Allow normal re-login on same device
Maintain strict control across devices

Always use set_password

Always use Django ORM

Security Philosophy

The system is designed to:

Reduce account sharing
Allow normal user behavior (same device re-login)
Enforce strict device/session control
Maintain auditability (no hard deletes)
